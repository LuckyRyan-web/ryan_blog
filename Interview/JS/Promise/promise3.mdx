### 1

```ts
const promise = new Promise((resolve, reject) => {
    resolve('success1')
    reject('error')
    resolve('success2')
})
promise
    .then((res) => {
        console.log('then: ', res)
    })
    .catch((err) => {
        console.log('catch: ', err)
    })
```

output:

```
"then: success1"
```

resolve 或者是 reject 只有第一次有效

### 2

```ts
const promise = new Promise((resolve, reject) => {
    reject('error')
    resolve('success2')
})
promise
    .then((res) => {
        console.log('then1: ', res)
    })
    .then((res) => {
        console.log('then2: ', res)
    })
    .catch((err) => {
        console.log('catch: ', err)
    })
    .then((res) => {
        console.log('then3: ', res)
    })
```

output:

```
catch: error
then3: undefined
```

catch 不管被连接到哪里都可以捕捉到之前没有捕捉到的错误

catch 也会返回一个 promise 只是没有返回值，所以 then 会返回一个 undefined

### 3

```ts
Promise.resolve(1)
    .then((res) => {
        console.log('res 1:', res)
        return 2
    })
    .catch((err) => {
        return 3
    })
    .then((res) => {
        console.log('res 2:', res)
    })
```

output:

```
res 1: 1
res 2: 2
```

promise 链式调用，每次的 then 和 catch 都会返回一个新的 Promise，所以第一个输出的 `res 1: 1` 之后的 `return 2` 其实也就是 `resolve(2)`，所以会出现第二个返回 `res 2: 2`

### 4

```ts
Promise.reject(1)
    .then((res) => {
        console.log('res 1:', res)
        return 2
    })
    .catch((err) => {
        console.log('err 1:', err)
        return 3
    })
    .then((res) => {
        console.log('res 2:', res)
    })
```

output:

```
err 1: 1
res 2: 3
```

因为 reject 走的是 catch

### 5

```ts
const promise = new Promise((resolve, reject) => {
    setTimeout(() => {
        console.log('timer')
        resolve('success')
    }, 1000)
})

const start = Date.now()

promise.then((res) => {
    console.log(res, Date.now() - start)
})

promise.then((res) => {
    console.log(res, Date.now() - start)
})
```

output:

```
timer
1000
1000
```

then 可以执行多次，但是 new Promise 只执行一次，且值是固定的，后续的 then 和 catch 拿到的值都是一样的

### 6

```ts
Promise.resolve()
    .then(() => {
        return new Error('error!!!')
    })
    .then((res) => {
        console.log('then: ', res)
    })
    .catch((err) => {
        console.log('catch: ', err)
    })
```

output:

```
"then: " "Error: error!!!"
```

then resove 了一个 Error 给了 下一个 then， then 的 return 会返回一个 `resolve(new Error('error!!!'))`

### 7

```ts
const promise = Promise.resolve().then(() => {
    return promise
})
promise.catch('err', console.err)
```

会报死循环的错

### 8

```ts
Promise.resolve(1).then(2).then(Promise.resolve(3)).then(console.log)
```

then 或者是 catch 参数期望值是函数，如果不是函数的话就会透传，所以只会传递参数 1

### 9

```ts
Promise.reject('err!!!')
    .then(
        (res) => {
            console.log('success', res)
        },
        (err) => {
            console.log('error', err)
        }
    )
    .catch((err) => {
        console.log('catch', err)
    })
```

output:

```
'catch' 'error!!!'
```

```ts
Promise.resolve()
    .then(
        function success(res) {
            throw new Error('error!!!')
        },
        function fail1(err) {
            console.log('fail1', err)
        }
    )
    .catch(function fail2(err) {
        console.log('fail2', err)
    })
```

这里要注意第一个参数是 onFulfilled、第二个参数是 onRejected

promise.resolve() 会执行第一个参数的函数，但是第一个会抛出错误，所以不会被第二个参数执行，会直接被 catch 捕获到

### 10

```ts
Promise.resolve('1')
    .then((res) => {
        console.log(res)
    })
    .finally(() => {
        console.log('finally')
    })

Promise.resolve('2')
    .finally(() => {
        console.log('finally2')
        return '我是finally2返回的值'
    })
    .then((res) => {
        console.log('finally2后面的then函数', res)
    })
```

output:

```
1
finally2
finally
finally2后面的then函数 2
```

```ts
function promise1() {
    let p = new Promise((resolve) => {
        console.log('promise1')
        resolve(1)
    })
    return p
}
function promise2() {
    return new Promise((resolve, reject) => {
        reject('error')
    })
}
promise1()
    .then((res) => console.log(res))
    .catch((err) => console.log(err))
    .finally(() => console.log('finally1'))

promise2()
    .then((res) => console.log(res))
    .catch((err) => console.log(err))
    .finally(() => console.log('finally2'))
```

output:

```
'promise1'
'1'
'error'
'finally1'
'finally2'
```

首先要执行同步任务先，也就是先执行两个 promise 的构造函数

1. 打印 promise1
2. resolve(1) 意味着 .then() 要放在下一个微任务队列
3. 执行 reject('error') 需要把 catch 任务放到下一个队列
4. 执行 console.log(1)
5. 执行 console.log('error')
6. 然后遇到 finally1 和 finally2 此时微任务队列里面已经没有其他任务了，顺序执行这两个
